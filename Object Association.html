Usage Guidelines & Best Practices

Implementation guidance for Object Association cards

When to Use Small Cards

  • Dense relationship lists with 10+ associations
  • Narrow side panels or drawers
  • Quick-scan contexts where detail isn't needed
  • Mobile or responsive layouts

When to Use Medium Cards

  • Default relationship panels in main content area
  • Balanced information density needed
  • When users need to understand relationship context
  • Grid layouts with 2-4 columns

When to Use Large Cards

  • Inspector panels and detail views
  • When full relationship metadata is required
  • Expanded preview states
  • Administrative or audit contexts

Cross-Variant Consistency Rules

Visual Consistency

All variants must use consistent iconography, color coding for association types, and directional indicators.

Relationship Direction

Source vs target distinction must be clear across all card sizes using consistent visual patterns.

Component Alignment

Object Association cards should visually align with Associated Tag and Audit cards in the design system.

Permission Awareness

All cards must gracefully handle permission-restricted content without exposing internal IDs or sensitive data.

Association Status States

Visual representation of different association lifecycle states

Association
Active
Fully active and operational relationship
Most common state for working associations

Requires administrative action
Association

Association

Historical record, no longer active
Preserved for audit and compliance

State Transition Rules

Pending → Active

Requires approval from authorized user or automatic validation rules

Active → Inactive

Can be toggled by users with appropriate permissions, reversible action

Any State → Archived

Permanent state change, typically irreversible, preserves historical record
