Interaction Patterns

Consistent behavior patterns across all card variants ensuring predictable user experience.

Primary Click Action

All card sizes support full-card click to open complaint details in main view.

onClick → navigateToComplaint(id)

Hover States

Cards show subtle elevation and border color change on hover to indicate interactivity.

hover:border-gray-300
hover:shadow-md

Keyboard Navigation

All cards are keyboard accessible with focus states and Enter key activation.

Enter → Activate

Accessibility

WCAG 2.1 AA compliant design patterns ensuring inclusive user experience.

Semantic HTML

  • Proper heading hierarchy (h1-h4)
  • Landmark regions (article, section)
  • Button elements for interactions
  • Lists for grouped content

Color Contrast

  • 4.5:1 minimum for body text
  • 3:1 minimum for large text
  • Status badges meet contrast requirements
  • Icons paired with text labels

Keyboard Navigation

  • Tab order follows visual layout
  • Focus indicators clearly visible
  • Enter/Space activate buttons
  • Escape closes menus/modals

Screen Reader Support

  • ARIA labels for icons
  • ARIA live regions for updates
  • Descriptive link text
  • Status announcements
