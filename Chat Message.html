Chat Message Object Cards

Chat Message cards represent individual messages within a Chat Conversation, authored by a user or an AI/system agent. These cards emphasize who said what and when, optimized for chronological scanning and conversation context without conversation management or editing capabilities.

Small Card (28.1)

Dense timelines, compact views, rapid scanning

Medium Card (28.2)

Default conversation views, balanced readability

Large Card (28.3)

Inspector panels, full context, detailed metadata

Design Principles

Core principles governing Chat Message card design across all variants

Clear Authorship

Every message must clearly identify whether it was authored by a user or an AI/system assistant

Temporal Context

Timestamps must be visible and consistent across all card sizes for chronological navigation

Scalable Hierarchy

Information density scales progressively from Small to Large while maintaining consistent structure

Read-Only Focus

Cards display message content without editing, deletion, or conversation management affordances

Small Card Specifications

Required Elements

  • Author indicator (avatar or role icon)
  • Short message preview (1–2 lines, clamped)
  • Compact timestamp
  • Author name or role label

Behavioral Notes

  • No actions visible by default
  • No related object context shown
  • Entire card clickable to expand
  • Minimal vertical height for density

Medium Card Specifications

Required Elements

  • Author avatar or role icon + object type label
  • Author name or role (User / Assistant)
  • Message content (clamped to 4 lines)
  • Timestamp (relative format)
  • Primary action (View in Conversation)

Optional Elements

  • Action type indicator (if message triggered action)
  • Message content remains readable but constrained
  • Author identity visually clear and prominent
  • Actions minimal and predictable

Large Card Specifications

Required Elements

  • Author avatar or role icon + object type label
  • Author name or role (prominent display)
  • Full message content (read-only, no truncation)
  • Comprehensive metadata grid
  • Parent conversation reference
  • Action bar with multiple actions

Behavioral Notes

  • Content must be read-only (no editing)
  • No message deletion affordances
  • Metadata remains secondary to content
  • Actions must be permission-aware
  • Support for action indicators and triggers

Cross-Variant Consistency Rules

Essential principles ensuring visual and functional coherence across all card sizes

Visual Alignment

Chat Message cards must align visually with Note and Audit cards, maintaining consistent spacing, typography, and structural patterns across the entire ArkCase design system.

Placement Consistency

Author and timestamp placement must be consistent across all sizes. Users should never need to search for these critical elements regardless of card variant.

Progressive Disclosure

Message content visibility should scale gradually from Small to Large. Each size reveals progressively more information without overwhelming the interface.

Component Cohesion

All variants should feel like scaled versions of the same component. Users should instantly recognize a Chat Message card regardless of its size or context.

Design System Integration

These consistency rules ensure that Chat Message cards integrate seamlessly with other ArkCase object cards (Notes, Audits, Tasks, Documents) while maintaining their unique identity and purpose within the conversation context.

Visual Hierarchy Principles

Color Coding

Assistant messages use blue accents to distinguish AI-generated content from human-authored messages. System messages use neutral gray to indicate automated actions without visual hierarchy competition.

Icon Consistency

User messages display profile photos, Assistant messages show a robot icon, and System messages use a gear icon. This iconography remains consistent across all card sizes and contexts.

Design Constraints & Limitations

Critical boundaries that must be respected in Chat Message card implementation

No Conversation Management

Do not design chat input fields, reply interfaces, or conversation creation tools within Chat Message cards. These cards are for display only.

No System Prompt Exposure

Never expose raw system prompts, internal logs, or sensitive AI configuration details in message content or metadata.

No Deletion Assumptions

Do not assume message deletion permissions. Deletion affordances should only appear when explicitly permitted by system rules.

No Moderation Interfaces

Chat Message cards focus strictly on message content, authorship, and timing. Do not include moderation, flagging, or content management tools.
