Small Card Specifications

Behavioral Notes
  • No message previews
  • No participant lists
  • Entire card clickable
  • Minimal vertical height

  • Related object visually secondary
  • Minimal, predictable actions

Large Card Specifications

Required Elements
  • Chat icon + object type label
  • Conversation title (prominent)
  • Status indicator (active/archived)
  • Metadata grid: Created date, Last activity, Initiating user, Conversation ID
  • Related object section (type + title/number)
  • Participant summary (user + AI/system)
  • Action bar with multiple actions
Behavioral Notes
  • Do not display full message history
  • Participant details should be summarized
  • Actions must be permission-aware
  • Strong visual hierarchy
  • Clear separation of identity, context, and actions

Cross-Variant Consistency Rules

Visual Alignment

Chat Conversation cards must align visually with Notification and Task cards, maintaining consistent design language across the ArkCase system.

Timestamp Placement

Timestamp placement should be consistent across all card sizes, always appearing in predictable locations for quick scanning.

Context Labeling

Context labeling should scale gradually from Small to Large variants, progressively revealing more information as card size increases.

Scaled Components

All variants should feel like scaled versions of the same component, maintaining recognizable patterns and relationships.

Design Constraints & Guidelines

Do Not Include

  • Chat input or messaging interfaces
  • Full message history displays
  • Model or token-level technical details
  • Message deletion or editing controls
  • Global navigation or page headers

Focus Areas

  • Conversation context and relevance
  • Recency and activity indicators
  • Related object associations
  • Participant identification
  • Navigation to full conversation

Usage Guidelines

Typical scenarios where Chat Conversation cards are displayed within ArkCase.

Conversation Lists

Display all active and archived conversations in a scrollable list view, using Small or Medium cards for density.

Inspector Panels

Interaction Patterns

Clickable Card Areas

In Small and Medium variants, the entire card should be clickable to open the conversation. In Large variants, specific action buttons provide clear navigation paths.

Small: Full Card Medium: Full Card Large: Button Actions

Hover States

Cards should provide visual feedback on hover with subtle border color changes and shadow elevation to indicate interactivity.

Border Highlight Shadow Elevation

Keyboard Navigation

Cards must be keyboard accessible with clear focus indicators. Tab navigation should move logically through card elements and action buttons.

Tab Navigation Focus Indicators Enter to Activate

Contextual Actions

Large cards include a more menu (⋮) for additional actions such as archiving, sharing, or marking as unread. Actions must respect user permissions.

Permission-Aware Contextual Menu

Accessibility Requirements

Visual Accessibility

  • Minimum 4.5:1 contrast ratio for text
  • Status not conveyed by color alone
  • Clear visual focus indicators
  • Sufficient spacing for touch targets

Screen Reader Support

  • Semantic HTML structure
  • Descriptive ARIA labels for icons
  • Meaningful link and button text
  • Status announcements for updates

Responsive Behavior

Desktop (1024px+)

All three card variants (Small, Medium, Large) are available. Grid layouts support multiple columns. Inspector panels can show Large cards alongside main content.

Tablet (768px - 1023px)

Medium and Large cards adapt to single or two-column layouts. Small cards may stack in narrow sidebars. Touch targets are enlarged for better interaction.

Mobile (< 768px)

Cards stack vertically in single-column layouts. Small cards remain compact while Medium cards expand to show key information. Large cards may be simplified or shown in full-screen modals.

Implementation Notes

Data Requirements

Each Chat Conversation card requires the following data points:

conversationId Unique identifier
title Conversation name/context
status Active, Archived, etc.
lastActivity Timestamp of last message
relatedObject Associated case/task/complaint
participants Array of user/AI participants
createdDate Conversation creation timestamp
initiatedBy User who started conversation

Component Integration

Chat Conversation cards should integrate seamlessly with existing ArkCase components:

  • Share visual styling with Notification and Task cards
  • Use common timestamp formatting utilities
  • Leverage existing related object link components
  • Implement permission checks for action visibility

Performance Considerations

  • Implement virtual scrolling for long conversation lists
  • Lazy load participant avatars and related object details
  • Cache conversation metadata to reduce API calls
  • Use skeleton loaders during initial data fetch
