Design Guidelines

Technical specifications and consistency rules for implementing Cost cards across the ArkCase platform.

Visual Hierarchy

  • Amount Prominence
    Cost amount must be the most visually prominent element in all variants
  • Object Type Clarity
    Cost icon and label must clearly identify the object type
  • Progressive Disclosure
    Information density scales from Small → Medium → Large

Typography Scale

  • Small Card Amount
    text-lg font-bold (18px, 700 weight)
  • Medium Card Amount
    text-xl font-bold (20px, 700 weight)
  • Large Card Amount
    text-3xl font-bold (30px, 700 weight)

Spacing & Layout

  • Small Card Padding
    px-4 py-3 (16px horizontal, 12px vertical)
  • Medium Card Padding
    p-5 (20px all sides)
  • Large Card Padding
    p-6 (24px all sides)

Color System

  • Cost Icon Background
    bg-blue-100 (light blue, #DBEAFE)
  • Cost Icon Color
    text-blue-600 (blue, #2563EB)
  • Parent Reference
    bg-blue-50 with border-blue-100

Interaction States

  • Hover State
    shadow-md or shadow-lg with smooth transition
  • Clickable Cards
    cursor-pointer on Small cards, actions visible on Medium/Large
  • Button States
    hover:bg-blue-700 for primary actions

Accessibility

  • Color Contrast
    Minimum WCAG AA compliance for all text
  • Keyboard Navigation
    All interactive elements must be keyboard accessible
  • Screen Reader Support
    Semantic HTML and ARIA labels where needed

Consistency Rules

Essential rules for maintaining visual and functional consistency across all Cost card variants.

Costsheet Alignment

Cost cards must align visually with Costsheet cards to maintain system coherence. Use consistent iconography, color schemes, and structural patterns.

Implementation Note:
Both objects should use the same blue color palette and share similar card structure patterns

Currency Formatting

All currency amounts must follow consistent formatting rules across variants. Use standard locale-specific formatting with appropriate decimal places.

Format Pattern:
$X,XXX.XX USD (include currency symbol, thousand separators, two decimal places)

Contextual Scaling

Context should scale gradually from Small to Large variants. Small cards show minimal context, Medium adds parent reference, Large provides comprehensive metadata.

Scaling Principle:
Each variant should feel like a scaled version of the same component, not a different design

Permission Awareness

All cards must gracefully handle permission-restricted scenarios. Actions and data may be hidden, disabled, or redacted based on user permissions.

Security Note:
Never expose restricted financial data. Implement proper authorization checks before rendering sensitive information

Implementation Notes

Technical considerations and best practices for implementing Cost cards in production.

Required Elements

  • Cost icon with consistent styling
  • Formatted cost amount with currency
  • Cost title or description
  • Cost date (when incurred)
  • Parent Costsheet reference (Medium/Large)

Avoid Including

  • Approval or reimbursement workflows
  • Financial calculation details
  • Line-item editing interfaces
  • Accounting system integrations
  • Global navigation or page headers

Key Design Principles

Amount First
Cost amount should be the most prominent visual element in every variant
Clear Context
Users must immediately understand what, when, and which Costsheet
Scannable Layout
Optimize for quick scanning in lists and rapid information retrieval
