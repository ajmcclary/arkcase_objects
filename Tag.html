Large Card Specifications

  • Layout: Structured card with clear visual sections and strong hierarchy
  • Required Elements: Icon + label, prominent tag name, full description, metadata grid, associations section, action bar
  • Metadata: Usage count, created date, and other relevant timestamps
  • Associations: Summarized by object type with counts (not individual items)
  • Actions: Multiple actions, permission-aware (View Usage, More)
  • Context: Inspector panels, expanded previews, detail views

Accessibility Considerations

Design principles for keyboard navigation and screen reader support

Keyboard Navigation

  • Tab: Move focus between tag cards
  • Enter/Space: Select or activate tag
  • Arrow Keys: Navigate within tag lists
  • Escape: Close tag picker or deselect

Screen Reader Support

  • Announce "Tag: [name]" for each card
  • Include usage count in announcement
  • Indicate selection state clearly
  • Provide context for tag descriptions

Implementation Best Practices

Guidelines for proper tag card usage and implementation

Do

  • Use consistent tag iconography across all variants
  • Display usage counts to indicate tag popularity
  • Clamp long descriptions in medium cards (2 lines max)
  • Summarize associations by object type, not individual items
  • Provide clear visual feedback for selection states
  • Make entire small cards clickable for better UX
  • Use color coding consistently for tag categories

Don't

  • Show individual tagged objects in large cards
  • Display internal IDs or taxonomy rules to users
  • Include tag creation interfaces in card components
  • Assume administrative permissions for all users
  • Use visual density heavier than Note cards
  • Truncate tag names in any variant
  • Mix different visual styles within same context

Responsive Behavior

How tag cards adapt across different screen sizes

Desktop (1024px+)

  • Full card layouts with all elements visible
  • Multi-column grids for medium cards (2-3 columns)
  • Side-by-side associations in large cards

Tablet (768px - 1023px)

  • Two-column layout for medium cards
  • Slightly reduced padding and spacing
  • Single column for large cards

Mobile (< 768px)

  • Single column layout for all card variants
  • Stacked associations in large cards
  • Full-width buttons in action bars

Summary

Key takeaways for implementing tag card components

Small Cards

Optimized for dense contexts like filters and pickers. Minimal footprint with icon and name only. Entire card is interactive.

Medium Cards

Default representation balancing identity with context. Includes description and usage count. Primary action for quick access.

Large Cards

Comprehensive view for inspector panels. Full description, metadata, and object type associations. Multiple actions available.

Core Principles

Consistent visual identity across all variants
Clear communication of usage and context
Permission-aware rendering throughout
Accessible keyboard and screen reader support
