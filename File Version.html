File Version Cards

Document version history and tracking

These cards represent individual versions of files within ArkCase. Each variant shows version numbers, change authors, and modification timestamps for document version control.

Optimized for dense version lists and history panels. Minimal vertical height, single-row layout.

Consistency Rules

1

Visual Hierarchy

File Version cards must visually nest under File cards through reduced visual weight, smaller dimensions, and subordinate styling cues.

2

Version Labeling

Version labels and numbers must be consistent across all card sizes, using the same format and prominence relative to other elements.

3

Active Indicators

Active/current version indicators must be visually distinct and consistently styled across all variants using color, badges, or borders.

4

Scalable Components

All variants should feel like scaled versions of the same component, maintaining proportional relationships between elements.

Design Guidelines

  • Do not design diff or comparison interfaces within these cards
  • Do not expose raw CMIS or repository identifiers to users
  • Do not assume version deletion permissions are available
  • Focus strictly on version identity and metadata presentation
  • Hash or integrity data must be summarized, not displayed in raw format
