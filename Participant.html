Design Guidelines

Core principles governing participant card design and implementation

Visual Hierarchy

Participant cards are always subordinate to their parent object. Identity and role information takes priority over access details.

  • Clear role identification
  • Parent context visible
  • Access level summarized

Cross-Object Consistency

Participant cards maintain consistent structure across all object types while adapting to contextual requirements.

  • Unified visual language
  • Predictable layouts
  • Scalable patterns

Role Clarity

Every participant card must immediately communicate who, what role, and level of access without requiring additional context.

  • Instant recognition
  • Minimal cognitive load
  • Self-documenting

Responsive Behavior

How Participant cards adapt to different screen sizes.

Variant Desktop (1024px+) Tablet (768-1023px) Mobile (<768px)
Small Inline with avatar, name, and role Inline layout maintained Compact single-line display
Medium Full participant details visible Contact info condensed Vertical stacking, avatar prominent
Large Full profile with activity history Tabs for sections Scrollable sections, collapsible areas

Accessibility

Keyboard navigation and screen reader support for Participant cards.

Keyboard Navigation

  • Tab: Navigate between participant cards
  • Enter/Space: Open participant profile or actions
  • Arrow Keys: Navigate within participant lists
  • Escape: Close dialogs or menus

Screen Reader Support

  • Semantic HTML with proper heading hierarchy
  • ARIA labels for role and participation type
  • Avatar images have meaningful alt text
  • Contact actions clearly labeled

Participant Card System Summary

Quick reference for Participant card variants and usage.

Small Cards

Compact references in participant lists, mentions, and assignment displays.

Medium Cards

Case participant panels, team views, and role assignment interfaces.

Large Cards

Participant detail views, activity history, and full profile display.

Core Principles

Consistent visual identity across sizes
Clear role and access level indication
Accessible keyboard and screen reader support
Responsive behavior across breakpoints
