Responsive Behavior

How Costsheet cards adapt to different screen sizes.

Variant Desktop (1024px+) Tablet (768-1023px) Mobile (<768px)
Small Inline layout, full currency display Inline layout maintained Compact row, abbreviated currency
Medium Full financial summary visible 2 column grid Single column, totals prioritized
Large Complete financial breakdown visible Condensed line items Scrollable line items, summary fixed

Accessibility

Keyboard navigation and screen reader support for Costsheet cards.

Keyboard Navigation

  • Tab: Navigate between costsheet cards
  • Enter/Space: Open costsheet details
  • Arrow Keys: Navigate within cost item lists
  • Escape: Close modals or deselect

Screen Reader Support

  • Currency values read with proper formatting ($X,XXX.XX)
  • Status badges announced with context
  • Financial totals clearly labeled
  • ARIA labels for icons and actions

Design Guidelines

Constraints and focus areas for Costsheet card design.

Prohibited Elements

  • Do not allow inline editing of financial values
  • Do not display unformatted currency values (use $X,XXX.XX)
  • Do not show partial totals without clear context

Design Focus Areas

  • Financial accuracy is paramount—always display verified totals
  • Use emerald color family for financial indicators
  • Maintain consistent currency formatting throughout

Consistency Rules

Design principles ensuring Costsheet cards align with the broader design system.

Visual Alignment

Costsheet cards align visually with Cost cards, using emerald icons and consistent financial formatting.

Progressive Disclosure

Small shows total only, Medium adds line item count, Large shows full breakdown.

Status Consistency

Status badges use consistent colors: blue (active), yellow (pending), green (approved), red (rejected).

Currency Formatting

Always use $X,XXX.XX format with proper thousand separators and two decimal places.

Implementation Notes

Technical considerations for Costsheet card development.

Component Architecture

  • Use semantic HTML for financial tables
  • Implement currency formatter utility function
  • Support locale-aware number formatting

Performance

  • Lazy load cost item details on expand
  • Virtualize long lists of line items
  • Cache calculated totals when possible
