Large Card Specifications

Required Elements

  • Organization icon + type label
  • Organization name (prominent)
  • Organization type and status
  • Restriction indicator
  • Contact section (email + phone)
  • Addresses (summarized)
  • Metadata grid
  • Associations section
  • Action bar with multiple actions

  • DBA names (summarized)
  • Parent/child relationships
  • Secondary contact methods
  • Additional metadata fields
  • More actions dropdown

Behavioral Notes

  • Addresses must be summarized
  • Relationships as counts/labels
  • DBA names don't dominate layout
  • Actions are permission-aware
  • Strong visual hierarchy

Interaction Guidelines

Hover: border-gray-300 + shadow-md transition

Selected: bg-blue-50 + border-2 border-blue-500

Focus: ring-2 ring-blue-500 ring-offset-2

Disabled: opacity-60 + grayscale + cursor-not-allowed

Loading: animate-pulse + gray-200 shapes

Transitions: 150ms ease-in-out for all states

Consistency Rules

Visual Alignment

Organization cards must align visually with Person and User cards across the entire system

Name Consistency

Name placement and styling should be consistent across all card sizes and variants

Progressive Disclosure

Contact and hierarchy information should scale gradually from Small → Medium → Large variants

Scaled Components

All variants should feel like scaled versions of the same component with consistent design language

Usage Guidelines

Small Card Context

Ideal Use Cases

  • Organization picker dialogs
  • Related organization lists
  • Search result previews
  • Dense association panels

Layout Constraints

  • Maximum height: 60px
  • Responsive width (100%)
  • Single-line text truncation

Medium Card Context

Ideal Use Cases

  • Organization directory lists
  • Case-related organization panels
  • Dashboard organization widgets
  • Organization comparison views

Layout Constraints

  • Typical height: 180-220px
  • Minimum width: 320px
  • Multi-line text support

Large Card Context

Ideal Use Cases

  • Inspector panel previews
  • Detailed organization modals
  • Organization profile summaries
  • Relationship detail views

Layout Constraints

  • Typical height: 500-700px
  • Minimum width: 600px
  • Full content display

Accessibility

Keyboard Navigation

  • All interactive elements must be keyboard accessible
  • Tab order follows logical reading sequence
  • Enter/Space activates primary actions
  • Escape closes modals and menus

Screen Reader Support

  • All icons paired with descriptive text or ARIA labels
  • Status badges announced with role and state
  • Card structure uses semantic HTML
  • Relationship context clearly announced

Visual Indicators

  • Focus states clearly visible with 2px outline
  • Hover states provide visual feedback
  • Color contrast meets WCAG AA standards
  • Status communicated through color and text

Touch Targets

  • Minimum touch target size: 44x44px
  • Adequate spacing between interactive elements
  • Cards provide clear click/tap feedback
  • Action buttons easily distinguishable
