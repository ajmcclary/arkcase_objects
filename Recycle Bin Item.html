Usage Guidelines

Best practices and recommendations for implementing Recycle Bin Item cards in different contexts.

Do's

  • Always display the deleted badge on the item icon to clearly indicate recycle bin status
  • Show who deleted the item and when for accountability and audit trails
  • Include parent folder information to help users understand the item's original context
  • Use the large card variant for detailed inspector panels and decision-making contexts
  • Clearly distinguish between restore and permanent delete actions with color coding
  • Display countdown information for automatic purge schedules

Don'ts

  • Don't show file previews or content for deleted items
  • Don't expose storage identifiers or CMIS internal references
  • Don't enable permanent delete actions without proper permissions
  • Don't make permanent delete the primary action - restore should be emphasized
  • Don't show active file metadata like version history or collaboration details
  • Don't use recycle bin cards for items that haven't been deleted

Context-Specific Recommendations

Dense List Views
Use small cards for maximum information density in recycle bin overview lists
Dashboard Widgets
Use medium cards in grid layouts for recently deleted items dashboards
Inspector Panels
Use large cards when users need to review detailed provenance before recovery decisions
Permission-Restricted Views
Show appropriate warning banners and disable actions based on user permissions
