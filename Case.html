Usage Guidelines

Best practices for implementing case file cards

Do

  • Use small cards in dense lists and queues
  • Show status badges prominently
  • Truncate long titles appropriately
  • Group related metadata logically

Don't

  • Overcrowd small cards with too much info
  • Hide critical status indicators
  • Use inconsistent spacing between variants
  • Make clickable areas too small

Best Practices

  • Maintain visual hierarchy across sizes
  • Use color coding consistently
  • Test with real data and edge cases
  • Ensure responsive behavior works well
