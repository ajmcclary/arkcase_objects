Audit Event Cards

Immutable audit trail records for ArkCase actions

Cross-Variant Consistency Rules

Visual Alignment

Audit Event cards must align visually with Notification and Object History cards across the ArkCase design system. Icon styles, color schemes, and spacing should maintain consistency.

Timestamp Placement

Timestamp placement must be consistent across all card sizes. Small cards show relative time (e.g., "2m ago"), while Medium and Large cards display full timestamps with date and time.

Progressive Detail

Event descriptions should scale gradually from Small → Medium → Large. Small cards show minimal context, Medium cards add basic metadata, and Large cards provide comprehensive event details.

Component Scaling

All variants should feel like scaled versions of the same component. Icon containers, status badges, and action buttons should maintain proportional sizing and consistent visual language.

Prohibited Elements

Do not expose raw JSON payloads, diff data, or low-level log details. Do not include audit export interfaces, filtering controls, or administrative configuration options within card components.
