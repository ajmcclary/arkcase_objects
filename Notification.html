Component Overview

Notification cards represent system-generated or user-targeted notifications related to cases, tasks, documents, or other system events. These cards emphasize timing, relevance, and actionability.

Small Card (17.1)
Notification trays, dropdowns, activity feeds
Medium Card (17.2)
Default notification lists, balanced context
Large Card (17.3)
Inspector panels, expanded previews

Usage Guidelines & Best Practices

Implementation guidelines for notification card components.

Do

  • Use consistent timestamp formats across all card sizes
  • Maintain clear visual hierarchy between title and message content
  • Ensure read/unread indicators are immediately visible
  • Make entire small cards clickable for better usability
  • Clamp message previews in medium cards to 2 lines maximum
  • Show full context in large cards for inspector views

Don't

  • Don't expose raw system payloads or technical details
  • Don't include bulk management actions in individual cards
  • Don't make small cards taller than necessary
  • Don't show too many actions in medium cards
  • Don't allow notifications to become message threads
  • Don't assume delete permissions for all users

Accessibility Considerations

Ensuring notification cards are accessible to all users.

Keyboard Navigation

  • • Tab to focus cards
  • • Enter to activate
  • • Arrow keys for lists
  • • Escape to dismiss

Screen Readers

  • • Semantic HTML structure
  • • ARIA labels for icons
  • • Status announcements
  • • Meaningful link text

Visual Clarity

  • • WCAG AA contrast ratios
  • • Clear focus indicators
  • • Sufficient text sizing
  • • Color + icon redundancy
