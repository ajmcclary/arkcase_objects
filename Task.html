Design System Notes

Key principles and consistency rules for Task card implementation

Visual Hierarchy

  • Task title is always the primary visual element
  • Status badge positioned consistently across all variants
  • Due date and assignee information always visible
  • Parent object reference scales with card size

Cross-Object Consistency

  • Matches visual language of Case File, Complaint cards
  • Consistent color coding for status across object types
  • Unified icon system and typography
  • Predictable action button placement

Interaction Patterns

  • Entire card acts as primary click target
  • Hover states provide visual feedback
  • Actions scaled to card size and context
  • Permission-aware action visibility

Accessibility

  • WCAG AA compliant color contrast ratios
  • Icons paired with text labels
  • Keyboard navigation support
  • Screen reader friendly structure
